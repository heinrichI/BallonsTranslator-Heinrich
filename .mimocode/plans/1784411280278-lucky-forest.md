# Plan: Fix Font Size Mismatch After Pipeline Re-Run

## Problem

After pipeline Re-Run, block 7 shows 272px (204pt) in the font panel — the detector's original font size — even though the text is actually rendered at ~18.3pt. After user manually sets font to 10pt and switches pages, font reverts to 272px.

## Root Cause Trace (from log.txt)

### First pipeline run (lines 22-108): WORKS
1. `initTextBlock` → fontformat=204pt, `_saved_font_size=204pt`
2. Shrink via `FontAutoAdjuster.shrink()` → `setFontSize()` with `_is_auto_layout=False` → updates fontformat to 24.8pt
3. `_font_adjuster_sync` → fontformat=24.8pt ✅
4. `updateBlkFormat` → blk.fontformat=24.8pt ✅

### Pipeline Re-Run (lines 140-174): BROKEN
1. `initTextBlock` → fontformat=204pt, `_saved_font_size=204pt`
2. `_find_best_font_size()` → binary search finds optimal=18.3pt
   - Calls `setFontSize(mid, _is_auto_layout=True)` → **skips fontformat update** (textitem.py:1045)
   - `set_cursor_cfmt` changes doc_font to 18.3pt, but fontformat stays at 204pt
3. `_auto_layout` line 187: `fontformat.font_size = _saved_font_size` → **restores to 272px!**
4. `paint`: doc_font=18.3pt fontformat=272px → **MISMATCH**

### After user sets font to 10pt + page switch (lines 196-262): BROKEN
1. User sets 10pt → fontformat=13.3px, doc_font=10pt ✅ (consistent)
2. Page switch → `updateSceneTextitems` re-creates block
3. `initTextBlock`: blk.fontformat=10pt (correct), `_saved_font_size=10pt`
4. `_find_best_font_size` binary search → doc_font=18.3pt (fontformat stays 10pt)
5. `_auto_layout` line 187: `fontformat.font_size = _saved_font_size=10pt` → restores to 10pt
6. `paint`: doc_font=18.3pt fontformat=10pt → **MISMATCH** (different direction)

## The Bug

**`_auto_layout()` line 187** in `text_layout_manager.py`:
```python
blkitem.fontformat.font_size = blkitem._saved_font_size
```

This restores fontformat to the value captured in `initTextBlock`. But `_saved_font_size` is always the *input* font size (from `blk.fontformat`), not the *output* size found by binary search. After `_find_best_font_size` finds the optimal size and `_update_flow_layout` may adjust it, line 187 overwrites fontformat with the stale input value.

Additionally, `_find_best_font_size` uses `_is_auto_layout=True` which prevents `setFontSize` from updating fontformat — so fontformat never gets the optimal value in the first place.

## Fix

### Change 1: Sync fontformat after `_find_best_font_size`

**File**: `ui/text_layout_manager.py`, `_auto_layout()` (after line 163)

After `_find_best_font_size` returns `optimal_size`, update fontformat:

```python
optimal_size = self._find_best_font_size(
    blkitem, text, target_w, target_h, original_size
)

# NEW: Sync fontformat to optimal size found by binary search
if blkitem.fontformat is not None:
    blkitem.fontformat.font_size = pt2px(optimal_size)
if blkitem.blk is not None and blkitem.blk.fontformat is not None:
    blkitem.blk.fontformat.font_size = pt2px(optimal_size)
```

### Change 2: Remove `_saved_font_size` restore (line 183-187)

**File**: `ui/text_layout_manager.py`, `_auto_layout()` (delete lines 183-187)

Remove:
```python
# Restore font size — _font_adjuster_sync corrupts self.fontformat.font_size
# through shared state in FontFormat. Restore from the value set by initTextBlock
# (which reads from blk.fontformat — the canonical source of truth).
if hasattr(blkitem, '_saved_font_size') and blkitem.fontformat is not None:
    blkitem.fontformat.font_size = blkitem._saved_font_size
```

The `_font_adjuster_sync` concern is handled by `_update_flow_layout` which already syncs fontformat via `_font_adjuster_sync`. With Change 1 setting fontformat to optimal_size before `_update_flow_layout`, the sync during shrink/grow will correctly adjust fontformat to the final value.
