**Правило из mvp.txt:**
```
Model < Presenter > View (Interface) < View (PyQt)
```

View зависит ТОЛЬКО от Presenter Interface, НЕ от Model.

## Осталось сделать

### Исправление багов SelectionManager

**Проблема**: `RuntimeError: wrapped C/C++ object of type TransTextEdit has been deleted`

**Корень проблемы**: `SelectionManager.holding` ссылку на удалённый Qt виджет.

**Элегантное решение**: Подключиться к сигналу `destroyed` виджета и очищать ссылку при уничтожении:

```python
def change_hovering_widget(self, edit):
    if self.hovering_transwidget is not None and self.hovering_transwidget != edit:
        self.hovering_transwidget.setHoverEffect(False)
        # Отключаем старый сигнал
        try:
            self.hovering_transwidget.destroyed.disconnect(self._on_widget_destroyed)
        except:
            pass
    
    self.hovering_transwidget = edit
    
    if edit is not None:
        # Подключаем сигнал уничтожения
        edit.destroyed.connect(self._on_widget_destroyed)
        edit.setHoverEffect(True)

def _on_widget_destroyed(self):
    """Вызывается когда виджет уничтожен Qt."""
    self.hovering_transwidget = None
```

**Файлы для исправления:**
- `ui/selection_manager.py` — добавить `_on_widget_destroyed` метод, УДАЛИТЬ try-except блоки

## Архитектура MVP

```
View (PyQt) ──implements──▶ View (Interface) ◀──depends── Presenter ──depends──▶ Model

TextBlkItem ──implements──▶ ITextBlockView ◀──depends── TextBlockPresenter ──depends──▶ TextBlock
Canvas ──implements──▶ ISceneView ◀──depends── ScenePresenter ──depends──▶ TextBlock[]
FlowTextBlkItem ──implements──▶ IFlowBlockView ◀──depends── FlowPresenter ──depends──▶ TextBlock
```

**Правило**: View зависит ТОЛЬКО от Presenter Interface. Presenter зависит от Model и View Interface. Model не зависит ни от кого.
