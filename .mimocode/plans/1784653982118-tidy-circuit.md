# Fix: Flow control points reset after close/reopen

## Root cause

Two bugs in the save pipeline prevent control point changes from being persisted to disk:

### Bug 1: `projstate_unsaved` never set for reshape commands

When a user drags a control point handle:
1. `onTextBlkItemReshaped` → `scenetext_manager.push_undo_command(ReshapeItemCommand)`
2. `push_undo_command` (scenetext_manager.py:429-434) → `_undo_mgr.push_command(command)` (when `_undo_mgr` is set)
3. `UndoManager.push_command` pushes to the active stack but does NOT call `setProjSaveState(True)`
4. `projstate_unsaved` stays `False`
5. On close, `conditional_save()` checks `projstate_unsaved` → `False` → **skips saving entirely**

The old fallback path (`canvas.push_undo_command`) would have set `projstate_unsaved=True`, but it's never reached when `_undo_mgr` is set.

### Bug 2: `conditional_save` ignores shape-only changes

Even if `projstate_unsaved` were `True`, the save logic has a gap:
```python
update_scene_text = save_proj = self.canvas.text_change_unsaved()  # False for shape changes
save_rst_only = not self.canvas.draw_change_unsaved()             # True (no draw changes)
if not save_rst_only:  # False → save_proj stays False
    save_proj = True
```
Result: `saveCurrentPage(update_scene_text=False, save_proj=False)` → nothing is saved.

## Fix

### File 1: `ui/scenetext_manager.py` — line 429-434

In `push_undo_command`, when `_undo_mgr` is set, also mark the project as unsaved:

```python
def push_undo_command(self, command, update_pushed_step=True):
    if self._undo_mgr:
        self._undo_mgr.push_command(command)
        self.canvas.setProjSaveState(True)  # ← ADD THIS
    else:
        self.canvas.push_undo_command(command, update_pushed_step)
```

### File 2: `ui/mainwindow.py` — lines 648-657

In `conditional_save`, when `projstate_unsaved=True` but neither text nor draw changes are detected, still save the project. The `save_flow_points()` call during handle drags already updated `blk.left_points`/`blk.right_points` in memory, so `imgtrans_proj.save()` will write them correctly:

```python
def conditional_save(self, keep_exist_as_backup=False):
    if self.canvas.projstate_unsaved and not self.opening_dir:
        update_scene_text = self.canvas.text_change_unsaved()
        save_rst_only = not self.canvas.draw_change_unsaved()
        save_proj = update_scene_text or not save_rst_only
        # Shape/control point changes don't set textstep or drawstep,
        # but projstate_unsaved is True → always save the project JSON
        if not save_proj:
            save_proj = True
            save_rst_only = True

        self.saveCurrentPage(update_scene_text, save_proj, restore_interface=True,
                            save_rst_only=save_rst_only, keep_exist_as_backup=keep_exist_as_backup)
```

This ensures: when only control points changed, we still call `imgtrans_proj.save()` to persist the JSON.
