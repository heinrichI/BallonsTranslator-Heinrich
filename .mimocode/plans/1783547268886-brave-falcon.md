# Fix: Font size not shrinking when text blocks are resized

## Problem

When text blocks are made smaller (via resize handles), the font size doesn't reduce to fit the new dimensions. The `_auto_shrink_font()` loop in `flow_textitem.py` calls `setRelFontSize(factor)`, but the font doesn't actually shrink visually.

## Root Cause

The current `setRelFontSize` in `ui/textitem.py:944-970` iterates through **every text fragment** and multiplies each one's font size individually:

```python
while block.isValid():
    it = block.begin()
    while not it.atEnd():
        fragment = it.fragment()
        old_font_size = fragment.charFormat().fontPointSize()
        new_font_size = round(old_font_size * value, 2)
        # ... merge format per fragment
```

This approach has issues:
1. If fragments have different sizes, each gets multiplied independently — the relative proportions are preserved but the overall effect may not match expectations
2. After fragment-level changes, `reLayoutEverything()` is called which may not properly sync the layout's `_max_font_size` tracking
3. The layout may expand `available_height` during `reLayout()` to accommodate the text, causing the shrink loop to exit prematurely

## Fix (from `main-Refactor` branch)

Replace `setRelFontSize` with a simpler approach that uses the max font size as reference and applies a uniform size via `setFontSize`:

**File**: `ui/textitem.py`, lines 944-970

Replace the current implementation with:

```python
def setRelFontSize(self, value: float, repaint_background: bool = False, set_selected: bool = False, restore_cursor: bool = False, clip_size: bool = False, **kwargs):
    current = self.layout.max_font_size()
    if current < 1:
        current = self.document().defaultFont().pointSizeF()
    new_size = round(current * value, 2)
    self.setFontSize(new_size, repaint_background, set_selected, restore_cursor, clip_size, **kwargs)
```

This:
1. Gets the current max font size from the layout (single reference point)
2. Falls back to document default if layout hasn't computed yet
3. Calls `setFontSize()` which applies a uniform font size to the entire document and properly triggers `reLayoutEverything()`

## Why this works

- `setFontSize()` uses `set_cursor_cfmt()` which applies the format to the entire document uniformly
- It properly updates stroke padding if needed
- It calls `reLayoutEverything()` which correctly recalculates `shrink_height`
- The shrink loop in `_auto_shrink_font` then correctly sees the reduced `shrink_height` and converges
